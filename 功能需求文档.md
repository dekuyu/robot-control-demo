# YRC1000 机械臂 UDP 远程控制系统
## 功能需求文档（PRD）

**版本**：V1.0  
**日期**：2026-07-01  
**技术栈**：Vue3 + Python FastAPI + PostgreSQL  
**通信协议**：UDP（YERC 高速 Ethernet 协议）  
**控制对象**：安川 YRC1000 控制柜 / MOTOMAN 系列机械臂

---

## 1. 产品概述

### 1.1 背景

安川 YRC1000 控制柜提供高速 Ethernet（UDP）通信接口，允许上位机通过 IP 网络读写机器人内部变量（B变量、P变量、IO、寄存器等），并可实现远程控制机器人的运行状态。本系统旨在构建一套 **Web 化的机械臂远程监控与控制平台**，通过标准浏览器即可完成机械臂的多轴控制、速度调节、启停管理及安全保护。

### 1.2 通信架构

```
浏览器（Vue3 前端）
        ↕ HTTP/WebSocket
FastAPI 后端（Python）
        ↕ UDP（YERC 高速 Ethernet）
YRC1000 控制柜 LAN2 口
        ↕ 电源线
MOTOMAN 机械臂本体
```

**网络参数**（默认）：
- 机器人 IP：`192.168.255.1`
- 机器人端口：`10040`
- 通信模式：UDP，16进制数组收发，高低字节互换

### 1.3 目标用户

| 角色 | 职责 |
|------|------|
| 设备操作员 | 日常启停、监控机械臂状态 |
| 调试工程师 | 各轴单独控制、速度与坐标调试 |
| 系统管理员 | 账户管理、操作日志审计、安全配置 |

## 3. 详细功能需求

### F01 系统连接与实时状态监控

#### F01-1 网络连接配置

- 支持配置机器人 IP 地址、UDP 端口号
- 支持配置本地端口（0 = 任意端口）
- 连接前自动发送心跳包（读取 B000 变量）验证通路
- 连接状态持久化到数据库，下次启动自动恢复配置
- **连接超时**：3 次 UDP 发送无响应则判定为断开

#### F01-2 实时状态仪表板

**刷新频率**：≤ 20ms（通过 WebSocket 推送至前端）

实时展示以下信息：

| 数据项 | 读取命令 | 刷新方式 |
|--------|----------|----------|
| 机器人运行模式（示教/再现/远程） | 读取状态信息 | 轮询 |
| 伺服电源状态（ON/OFF） | 读取专用输入 | 轮询 |
| 各轴当前角度值（J1~J6/J7） | 读取机器人位置数据 | 轮询 |
| 末端笛卡尔坐标（X/Y/Z/Rx/Ry/Rz） | 读取机器人位置数据 | 轮询 |
| 各轴力矩百分比 | 读取力矩数据 | 轮询 |
| 各轴速度反馈值 | 读取位置偏差 | 轮询 |
| 当前报警状态 | 读取报警数据 | 轮询 |
| 当前执行程序名 | 读取执行程序信息 | 轮询 |

#### F01-3 连接状态指示

- 页面顶部常驻显示连接状态（绿色=已连接、红色=断开、黄色=重连中）
- 断开时自动重连，最大重连次数 5 次，超过后报警通知
- 所有控制按钮在断开状态下自动禁用

### F02 安全管理（最高优先级）

> ⚠️ **安全是本系统的首要原则**。所有控制功能必须建立在完善的安全机制之上。

#### F02-1 软件急停

- **急停按钮**：页面全局可见的红色醒目按钮（固定在界面右上角）
- 点击急停后：
  1. 立即发送急停 UDP 命令至控制柜
  2. 禁用所有控制操作按钮（恢复前不可再次启动）
  3. 在日志中记录急停触发时间、操作用户、触发原因
- **键盘快捷键**：`Ctrl + E` 触发软件急停（无需鼠标定位）
- 急停解除需要操作员在页面上**明确确认解除**，并输入原因说明

#### F02-2 操作前安全检查

执行任何运动控制前，系统必须完成以下自动检查（任一不通过则阻断操作）：

| 检查项 | 说明 |
|--------|------|
| 伺服电源确认 | 确认伺服已上电才允许下发运动指令 |
| 运行模式确认 | 仅在"远程模式"下允许远程运动指令 |
| 报警清零确认 | 当前存在未处理报警时，禁止运动 |
| 速度合规性 | 速度设定不得超过安全限制值（默认 50%，管理员可调整） |
| 操作人员二次确认 | 发送运动指令前弹出确认对话框（可配置是否启用） |

#### F02-3 软件限位保护

- 支持为每个轴设置软件限位（正向/负向角度范围）
- 下发坐标前自动检测目标值是否超出软件限位
- 超出限位时拒绝发送指令，并在界面给出明确提示
- 软件限位值独立于机器人硬件超程，作为额外保护层

#### F02-4 速度安全限制

- **系统最大速度上限**：默认 50%（S1 安全档），可由管理员调整至 100%
- 调试模式下建议最大速度不超过 10%
- 速度修改需在历史记录中可追溯

#### F02-5 安全互锁

- 机器人处于示教模式时，远程控制自动上锁（不允许任何运动指令）
- 伺服断电时，禁止下发运动指令
- 急停激活状态下，禁止任何启动操作

#### F02-6 操作日志强制记录

- 所有控制操作（运动指令、启停、急停、限位修改）必须写入数据库日志
- 日志包含：时间戳、操作用户、操作类型、参数值、执行结果
- 日志不可删除，保留 6 个月以上

### F03 机械臂启停控制

#### F03-1 伺服上电 / 断电

- **伺服上电**：向控制柜发送伺服使能命令
  - 上电前确认：安全模式为远程/再现模式
  - 上电前提示："请确认机器人运动范围内无人员"
  - 上电成功后状态指示灯变绿
- **伺服断电（正常停止）**：发送伺服断电命令
  - 机器人在当前位置停止（停止类别 2）
  - 断电后禁用所有运动控制

#### F03-2 程序启动 / 停止 / 暂停

- **启动程序**：发送 START 指令，可指定程序名称
- **停止程序**：发送 STOP 指令（停止类别 1 - 受控减速后停止）
- **暂停/继续**：发送 HOLD / 解除 HOLD 指令
- 所有操作在界面上有明确的状态反馈（执行中 / 暂停 / 已停止）

#### F03-3 复位操作

- 报警复位：清除当前报警状态，为重新启动做准备
- 复位后自动重新检查安全条件

### F04 多轴独立控制

#### F04-1 关节坐标系下的轴控制

支持机械臂各关节轴的独立点动控制，控制对象：

| 轴名 | 说明（6轴机器人） |
|------|-----------------|
| S 轴（J1） | 本体旋转 |
| L 轴（J2） | 下臂前后 |
| U 轴（J3） | 上臂上下 |
| R 轴（J4） | 下臂旋转 |
| B 轴（J5） | 手腕上下 |
| T 轴（J6） | 手腕旋转 |
| E 轴（J7） | 第 7 轴（可选，如有） |

**控制方式**：
1. **点动（JOG）控制**：按住方向按钮持续运动，松开立即停止
2. **增量移动**：输入步长值（如 1°、5°、10°），点击执行精确移动
3. **目标角度运动**：直接输入目标角度值，一键移动到指定位置

**实现方式**：通过写入 P 变量（目标坐标）+ 发送运动指令，或通过读取/写入 B 变量控制机器人 INFORM 程序中预设的运动逻辑

#### F04-2 直角坐标系下的轴控制

支持在笛卡尔坐标系下控制末端位置：

| 方向 | 说明 |
|------|------|
| X+ / X- | X 轴平移 |
| Y+ / Y- | Y 轴平移 |
| Z+ / Z- | Z 轴平移（垂直方向尤其注意） |
| Rx+ / Rx- | 绕 X 轴旋转 |
| Ry+ / Ry- | 绕 Y 轴旋转 |
| Rz+ / Rz- | 绕 Z 轴旋转 |

#### F04-3 多轴联动控制

- 支持同时设置多轴目标位置，一次性下发联动运动指令
- 目标点通过 P 变量写入（P000~P127）
- 运动前显示当前位置与目标位置的差值预览

#### F04-4 轴控制界面设计要求

- 6 轴（或 7 轴）以图形化示意图展示，点击对应轴进行选择
- 正向/负向各有独立的控制按钮，支持长按持续运动
- 当前轴角度以实时数字和进度条形式双重展示
- 接近软件限位时自动高亮警告提示

### F05 速度控制

#### F05-1 全局速度设定

- 提供速度滑块（0% ~ 100%，步进 1%）
- 实际发送速度 = 设定速度 × 全局速度上限系数（默认 50%）
- 速度变更后立即生效，影响后续所有运动指令

#### F05-2 分级速度档位

提供预设档位，快速切换：

| 档位 | 速度值 | 使用场景 |
|------|--------|----------|
| 微动档 | 1% ~ 5% | 精密定位调试 |
| 慢速档 | 10% ~ 20% | 初次运行验证 |
| 正常档 | 30% ~ 60% | 常规作业 |
| 高速档 | 70% ~ 100% | 已验证的自动化程序 |

#### F05-3 加减速控制

- 支持设置加速时间和减速时间（对应 INFORM 指令的 ACC/DEC 参数）
- 提供"柔性停止"选项（停止类别 1 - 受控减速）

#### F05-4 速度监控

- 实时显示各轴当前速度百分比（仪表盘样式）
- 速度超过设定安全阈值时，红色高亮警告

### F06 坐标与位置管理

#### F06-1 位置变量（P 变量）管理

基于 YERC UDP 协议，读写 P 变量（P000 ~ P127）：

- **读取 P 变量**：
  - 读取指定 P 变量的完整位置数据（X/Y/Z/Rx/Ry/Rz + 姿态 + 工具编号 + 坐标编号）
  - 支持批量读取（P000 ~ P127 全量读取）
  - 数据在界面以表格形式展示

- **写入 P 变量**：
  - 支持直接输入目标坐标值写入指定 P 变量
  - 支持将当前机器人位置"记录到"指定 P 变量
  - 写入时自动进行高低字节转换（×1000 后转 16 进制，再做大小端互换）

- **数值转换规则**（严格按照 YERC 协议）：
  ```
  原始浮点坐标值 3.11mm
  → 乘以 1000 得整数 3110
  → 转 16 进制 0x0C26
  → 高低位互换 0x260C
  → 写入报文字节数组
  ```

#### F06-2 位置点管理

- 支持将常用位置保存为"命名点位"（本地数据库存储）
- 命名点位包含：名称、描述、P 变量编号、各轴坐标值
- 支持点位的增删改查
- 支持一键"移动到"已保存的命名点位
- 支持点位导入/导出（JSON/CSV 格式）

#### F06-3 原点位置管理

- 显示当前各轴绝对原点数据
- 支持触发"移动到作业原点"操作
- 原点位置数据只读展示，不允许在线修改（需在示教编程器操作）

### F07 变量读写

#### F07-1 字节型变量（B 变量）读写

**B 变量范围**：B000 ~ B099

- **读取**：输入变量编号 → 发送读取命令 → 解析返回的 16 进制数据 → 显示十进制值
- **写入**：输入变量编号和目标值 → 发送写入命令 → 确认回复状态
- **批量读取**：一次性读取所有 B 变量，以表格展示
- **实时监视**：选择若干关注的 B 变量加入"监视列表"，以 20ms 频率自动刷新

**典型应用**（通过 B 变量控制 INFORM 程序逻辑）：
- B000 = 1 → 触发机器人程序中的某个动作分支
- B001 = 0 → 机器人暂停/等待状态标志

#### F07-2 位置型变量（P 变量）读写

（详见 F06-1，本节重申读写机制）

读取命令格式（16进制）：
```
59 45 52 43 20 00 00 00 03 01 00 00 00 00 00 00
39 39 39 39 39 39 39 39 7f 00 00 00 [变量编号2字节] 06 [元素编号] 00 00
```

- 元素编号 06 = 第 1 轴坐标（X）
- 元素编号 07 = 第 2 轴坐标（Y）
- 元素编号 08 = 第 3 轴坐标（Z）
- …以此类推

#### F07-3 IO 数据读写

- **通用输入（GI）**：读取通用输入信号状态（0/1）
- **通用输出（GO）**：读取/写入通用输出信号
- **界面展示**：以 LED 矩阵形式展示 IO 状态，可点击切换 Output 状态

#### F07-4 整数型变量（I 变量）/ 双精度变量（D 变量）

- 支持按需读写，提供通用的"变量调试面板"
- 支持十进制/十六进制双模式输入

### F08 报警与故障管理

#### F08-1 实时报警监控

- 轮询读取当前报警状态，有报警时在界面顶部红色横幅提示
- 显示报警编号、报警级别（重度/轻度）、发生时间
- 报警时所有运动控制自动禁用

#### F08-2 报警历史查询

- 读取 YRC1000 报警历史记录
- 支持按时间范围、报警级别筛选
- 同步存储到本地 PostgreSQL 数据库，保留历史记录

#### F08-3 报警复位

- 支持在报警处理完成后执行远程复位
- 复位前提示用户确认"故障原因已排除"

#### F08-4 报警通知（扩展）

- 预留邮件/Webhook 通知接口
- 支持配置关键报警的推送规则

### F09 操作日志与审计

#### F09-1 操作日志记录

每条日志包含以下字段：

| 字段 | 说明 |
|------|------|
| log_id | 唯一标识 |
| timestamp | 操作时间戳（毫秒精度） |
| user_id | 操作用户 ID |

- 支持按时间范围、操作类型、用户筛选
- 支持导出为 CSV 或 Excel 格式
- 管理员可查看全部用户日志，普通用户只能查看自己的操作日志

## F11 UDP/串口调试终端（新增）

> 为调试工程师和系统管理员提供一体化的 UDP 报文收发、解析与日志记录能力，替代传统桌面串口工具（如 ZLAN、SSCOM），并与现有机器人控制系统深度集成。

### F11-1 连接配置

- 支持配置工作模式（UDP，后续可扩展 TCP/串口）
- 支持配置本地端口（0 = 任意端口）
- 支持配置目标 IP 地址与目标端口（默认 `192.168.255.1:10040`）
- 连接前自动检测目标地址可达性，超时 3 秒无响应则判定为断开
- 连接状态持久化到数据库，下次启动自动恢复上次的终端配置
- 连接/断开按钮常驻，断开状态下发送按钮自动禁用

### F11-2 发送数据区

- 支持十六进制与字符串两种输入模式，一键切换
- 十六进制输入自动按字节分组（如 `59 45 52 43`），非法字符实时标红
- 支持快捷插入回车换行（`\r\n` / `\n` / `\r`）
- 支持自动发送（可配置间隔，默认 1000ms，最小 100ms）
- 支持发送历史与常用 YERC 命令模板（读状态、读位置、读 B 变量、读 P 变量等）
- 快捷键：`Ctrl + Enter` 发送，`Ctrl + L` 清空输入区

### F11-3 接收数据区

- 支持十六进制与字符串两种显示模式，一键切换
- 每条接收数据带时间戳（毫秒精度）和方向标识（← 接收 / → 发送）
- 十六进制显示按字节分组，发送与接收使用不同颜色高亮（如发送绿色、接收蓝色）
- 支持自动滚动、暂停滚动、一键清空、复制全部内容
- 接收缓冲区默认 2000 字节，超出时提示并可配置清空策略
- 断线或长时间无数据时，接收区顶部显示灰色提示

### F11-4 报文日志

- 每次发送/接收自动写入数据库 `packet_log` 表，包含：时间戳、方向、目标地址、原始十六进制、数据长度、操作用户
- 支持按时间范围、方向（发送/接收）、目标地址、操作用户筛选
- 支持导出为 CSV 或 Excel 格式
- 日志保留策略：热数据 7 天、温数据 30 天、冷数据归档 180 天（可配置）
- 报文日志不可删除，保留 6 个月以上

### F11-5 状态统计

- 页面顶部常驻显示：发送字节数、接收字节数、发送次数、接收次数、错误次数
- 支持一键重新计数
- 统计数据在本次会话中实时更新，断开连接后不清零（可手动重置）

### F11-6 权限与安全控制

- 仅管理员与调试工程师可发送 UDP 报文；操作员默认只读
- 发送前进行二次确认弹窗（可配置关闭，但建议保留）
- 默认限制发送频率 ≤ 10 次/秒，防止 UDP 报文风暴
- 对目标 IP 进行白名单校验，禁止向非机器人控制柜地址发送报文
- 危险命令（如未经验证的运动控制命令）建议通过 YERC 命令白名单拦截
- 所有终端操作写入操作日志，便于审计

### F11-7 数据库连接配置

- 本地开发默认数据库连接字符串：
  ```
  postgresql+asyncpg://robot_user:robot_123456@localhost:5432/robot_control
  ```
- 生产环境必须通过环境变量 `DATABASE_URL` 覆盖，禁止硬编码密码
- 系统启动时进行数据库连接健康检查，连接失败时页面顶部显示红色告警
- 数据库连接配置变更后需重启后端服务生效


### 4.1 YERC 报文格式

**请求报文（上位机 → YRC1000）**：

```
字节位置  说明
[0-3]    识别子：59 45 52 43（"YERC"）
[4-5]    标头部大小：20 00（固定 0x0020 = 32 字节）
[6-7]    数据部大小：可变
[8]      预约1：03
[9]      处理区分：01（读取）/ 00（写入）
[10-11]  预约1：00 00
[12-13]  ACK：01（请求时）
[14-15]  请求ID：00 00
[16-23]  数据块编号："99999999"（固定）
[24]     子标头命令编号
[25]     数据排列编号
[26]     单元编号
[27]     处理：00（读取）
[28+]    数据部：根据命令不同而变化
```

**应答报文（YRC1000 → 上位机）**：

```
[0-23]   同请求头格式
[24]     状态：0x80（成功）/ 0x81+（错误）
[25+]    返回数据
```

### 4.2 关键命令汇总

| 功能 | 命令代码 | 说明 |
|------|----------|------|
| 读取机器人状态 | 0x72（r）| 运行模式、伺服状态 |
| 读取机器人位置 | 0x75（u）| 各轴角度/末端坐标 |
| 读取报警信息 | 0x70（p）| 当前报警 |
| 读取 B 变量 | 0x7a | B000~B099 |
| 写入 B 变量 | 0x7a | B000~B099 |
| 读取 P 变量 | 0x7f | P000~P127 各轴坐标 |
| 写入 P 变量 | 0x7f | P000~P127 各轴坐标 |
| 读取 IO 信号 | 0x76（v）| 通用 IO |
| 写入 IO 信号 | 0x76（v）| 通用 IO |
| 读取力矩 | 0x74（t）| 各轴力矩百分比 |
| 读取轴构成 | 0x73（s）| 轴配置信息 |
| 控制命令（启动/停止） | 专用 | 再现模式运动控制 |

### 4.3 数值编解码规范

**浮点坐标值的编码**（上位机 → 机器人）：
1. 将坐标值 ×1000 取整数（保留 3 位小数精度）
2. 转换为 32 位有符号整数
3. 转换为 16 进制
4. **高低字节对调**（小端格式）
5. 填入报文数据部

**浮点坐标值的解码**（机器人 → 上位机）：
1. 读取数据部 4 字节
2. **高低字节对调**（还原为大端格式）
3. 转换为 32 位有符号整数
4. 除以 1000 得到实际坐标值（单位 mm / 度）

## 6. 非功能性需求

### 6.1 性能指标

| 指标 | 要求 |
|------|------|
| 状态数据刷新延迟 | ≤ 20ms（UDP 收发 + WebSocket 推送） |
| 控制指令响应时间 | ≤ 50ms（从点击到 UDP 发送完成） |
| 前端页面加载时间 | ≤ 2s |
| 并发连接数 | ≥ 5 个 WebSocket 同时监控 |
| 数据库写入性能 | ≥ 1000 条日志/秒 |

### 6.2 可靠性要求

- UDP 通信故障：自动重试 3 次，失败后记录错误并在界面报警
- 数据库故障：日志先写内存缓冲，数据库恢复后批量补写
- 系统崩溃恢复：重启后自动恢复连接配置和上次状态

### 6.3 安全要求

- 所有 API 接口需要 JWT 认证
- 控制类接口（写操作）需要额外的操作权限校验
- 传输层使用 HTTPS（生产环境）
- 数据库连接密码、JWT 密钥通过环境变量配置，不硬编码

### 6.4 兼容性要求

- 前端支持 Chrome 90+、Firefox 88+、Edge 90+
- 后端 Python 3.10+
- 支持 YRC1000 控制器固件版本（YERC 协议兼容）

## 8. 风险与约束

| 风险 | 影响 | 对策 |
|------|------|------|
| UDP 报文丢失 | 控制指令未执行 | 重要指令发送后验证回复，超时重试 |
| 网络抖动导致状态滞后 | 显示数据不准确 | 数据加时间戳，超过 100ms 未更新则灰显提示 |
| 机器人处于非远程模式 | 远程指令无效 | 发送前检查模式，不符合时拦截并提示用户 |
| 多用户同时操作 | 指令冲突 | 操作互斥锁，同一时刻只允许一个用户发送运动指令 |
| YERC 协议细节不符 | 通信失败 | 先开发单元测试，用调试工具验证每条命令的 16 进制报文 |
| 调试终端误发控制命令 | 机械臂意外运动或报警 | 权限分级 + 发送频率限制 + 危险命令白名单 + 二次确认 |
| 调试终端 UDP 报文风暴 | 网络拥塞 / 控制柜响应异常 | 默认发送频率 ≤ 10 次/秒，可配置上限 |
